## 1. Project Overview

The Portal Context Generator Module is part of a larger portal automation ecosystem. Its purpose is to create a **predefined knowledge base** for internal portals by ingesting documentation and user knowledge, then producing well-structured JSON files that describe:

- What a portal is and what it does
- Individual workflows and their step-by-step UI actions
- High-level UI interactions (click, select, input, navigate) — NOT DOM-level details

The tool uses an **LLM-powered iterative agent** that asks clarifying questions and refines the output based on user feedback, producing a solid, user-validated portal context.

## 2. Technology Stack

| Concern | Choice | Rationale |
|---|---|---|
| **Agent Framework** | **LangGraph** | Built for stateful, iterative, human-in-the-loop workflows. The `interrupt()` primitive lets the LLM pause, ask clarifying questions, and resume. |
| **LLM Providers** | **Multi-provider** | Gemini (via `langchain-google-genai`), Anthropic Claude (via `langchain-anthropic`), and locally-hosted OpenAI-compatible LLMs (via `langchain-openai` with custom `base_url`). |
| **UI** | **Streamlit** | Rapid Python-native UI with built-in file uploaders, chat interface, JSON display, and session state. |
| **Document Parsing** | **Custom parsers** | `pandas` for CSV/Excel, `BeautifulSoup` for HTML, `PyPDF2` for PDF, `python-docx` for Word, direct read for Markdown/Text. |
| **Persistence** | **SQLite** (LangGraph checkpointer) | Lightweight, zero-config. Stores conversation state for session continuity. |
| **Schema Validation** | **Pydantic v2** | Defines and validates the output JSON structure. Enables LLM structured output. |

## 3. Key Design Decisions

### 3.1 Multi-LLM Support

The application supports three LLM provider types:

1. **Google Gemini** — via `langchain-google-genai`, requires `GOOGLE_API_KEY`
2. **Anthropic Claude** — via `langchain-anthropic`, requires `ANTHROPIC_API_KEY`
3. **Local/OpenAI-compatible** — via `langchain-openai` with custom `base_url`, requires the URL and optionally an API key

The UI sidebar allows the user to select the provider and model, enter the appropriate credentials, and (for local LLMs) specify the base URL. Provider selection is validated before agent execution begins.

### 3.2 Split JSON Output

The tool generates **one JSON file per workflow/task**, not a single monolithic file. Additionally, a `_portal_info.json` file contains the top-level portal metadata. This makes individual workflows easy to review, version, and feed into downstream automation.

**Output structure:**
```
output/
├── _portal_info.json           # Portal metadata + UI elements
├── wf_001_submit_leave.json    # Workflow 1
├── wf_002_check_payslip.json   # Workflow 2
└── wf_003_update_profile.json  # Workflow 3
```

### 3.3 Export

The final approved JSON files are packaged as a **ZIP download**. Users can also download individual workflow JSON files.

### 3.4 No Authentication

Single-user tool. No login required. The user enters their LLM API key directly in the sidebar.

## 5. Architecture & Agent Flow

### 5.1 LangGraph Agent Flow

```
┌─────────┐     ┌──────────────────┐     ┌─────────────────┐
│  START   │────>│ Document         │────>│ Context          │
│          │     │ Processor        │     │ Builder          │
└─────────┘     └──────────────────┘     └────────┬────────┘
                                                   │
                                          ┌────────▼────────┐
                                          │ Question         │
                                          │ Generator        │
                                          └────────┬────────┘
                                                   │
                                          ┌────────▼────────┐
                                          │ INTERRUPT:       │
                                          │ Ask User         │◄──────┐
                                          └────────┬────────┘       │
                                                   │                │
                                          ┌────────▼────────┐       │
                                          │ JSON Generator   │       │
                                          └────────┬────────┘       │
                                                   │                │
                                          ┌────────▼────────┐       │
                                          │ INTERRUPT:       │       │
                                          │ Review JSON      │       │
                                          └────────┬────────┘       │
                                                   │                │
                                    ┌──────────────┴──────┐         │
                                    │                     │         │
                              User approves         User wants      │
                                    │               changes         │
                              ┌─────▼─────┐    ┌────────▼────────┐  │
                              │   END      │    │ JSON Refiner    │──┘
                              └───────────┘    └─────────────────┘
```

### 5.2 Node Descriptions

| Node | Responsibility |
|---|---|
| **Document Processor** | Parse uploaded files, extract text, create consolidated document summary |
| **Context Builder** | Use LLM to analyze documents, identify portal purpose, list workflows |
| **Question Generator** | LLM identifies gaps/ambiguities, generates targeted clarifying questions. Uses `interrupt()` to pause and ask the user. |
| **JSON Generator** | Produce structured workflow JSONs using LLM with `.with_structured_output()` |
| **JSON Refiner** | Accept user feedback, apply modifications to JSON. Loops back to review. |

### 5.3 Agent State

```python
class AgentState(TypedDict):
    documents: list[dict]            # Parsed document contents
    user_inputs: list[str]           # Free-text user inputs
    messages: Annotated[list, add_messages]  # Chat history
    portal_context: dict | None      # Portal info JSON
    workflows: list[dict]            # Generated workflow JSONs
    questions: list[str]             # Pending clarifying questions
    phase: str                       # "processing" | "questioning" | "generating" | "refining"
    iteration: int                   # Refinement iteration count
```
